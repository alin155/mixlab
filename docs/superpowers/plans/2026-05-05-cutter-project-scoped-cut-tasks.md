# 剪辑端项目级剪切任务归属计划

## 目标

把剪切任务、剪切任务页和素材定位页右侧任务栏收束到当前剪切项目下，避免所有项目共用一个全局任务队列。

## 当前问题

- 剪切项目已经记录搜索和剪切次数，但剪切任务仍按全局队列展示。
- API 返回的剪切任务没有项目字段，刷新后无法知道任务属于哪个项目。
- 剪辑师切换项目时，剪切任务页需要只看到当前项目的交付状态。

## 实施步骤

1. 在剪切任务状态层增加任务归属索引。
   - `CutQueueJob` 增加 `project_id` 和 `clip_list_id`。
   - 本地存储保存 `cut_job_id -> project_id` 与 `clip_list_id -> project_id` 映射。
   - API 队列刷新时根据索引补回项目归属。

2. 让创建剪切任务时写入当前项目归属。
   - 单段剪切和整清单剪切都要记录归属。
   - 本地模拟任务也带项目归属，便于测试和离线模式一致。

3. 页面只展示当前项目任务。
   - 素材定位页右侧任务栏使用当前项目队列。
   - 剪切任务页使用当前项目队列，并显示当前项目语境。
   - 无项目时不展示历史项目的任务。

4. 搜索记录按项目优先展示。
   - 在项目内搜索时，素材定位页左侧搜索记录来自当前项目。
   - 启动页继续展示全局最近搜索。

## 验收

- 创建剪切任务后，当前项目能看到任务。
- 切换项目后，剪切任务页不混入其他项目任务。
- API 刷新后任务归属不丢失。
- 单元测试覆盖任务归属索引、API 映射和页面项目语境。
