# M9C 管理端信息架构修正设计

## 背景

M9B 已经让管理端控件接入真实接口，但页面信息架构暴露出几个产品问题：

- 顶部路径展示没有可操作意义。
- 公共素材库设置页像只读展示，不像真正的配置中心。
- 设置页和公共素材库设置页职责重叠。
- 原视频封面接口返回的是相对地址，前端没有解析到管理端接口地址。
- 预处理队列缺少“哪些视频未处理、数量多少、操作会影响什么”的上下文。
- 健康诊断仍是工程报告语言，管理员难以判断检查目的和处理建议。
- 原视频列表缺少进入预处理详情的路径。

本阶段目标是修正管理端的产品结构和可理解性，再继续扩展功能。

## 总体原则

1. 管理端页面不出现英文界面文案。
   技术名词必须转成中文展示，例如：
   - `ready` 显示为“已可用”。
   - `queued` 显示为“队列中”。
   - `processing` 显示为“处理中”。
   - `unprocessed` 显示为“未处理”。
   - `failed` 显示为“处理失败”。
   - `index-required` 显示为“待发布索引”。
   - `Doctor` 显示为“健康诊断”。
   - `ASR` 显示为“语音转文字”。
   - `API Key` 显示为“接口密钥”。
2. 页面名称、按钮、表头、状态、阶段、提示、错误说明都必须中文化。
3. 技术详情可以保留真实路径、文件名和协议字段，但默认放在“技术详情”折叠区或详情页。
4. 管理端围绕“公共素材从哪里来、预处理产物存哪里、现在生产到哪一步、剪辑师能用多少”组织。
5. 浏览器页面不直接保存密钥，不直接启动长期后台工作进程；这些仍由本地服务或桌面壳承接。

## 导航结构

左侧导航调整为：

1. 仪表盘
2. 原视频管理
3. 预处理队列
4. 索引与发布
5. 健康诊断
6. 设置

删除独立的“公共素材库设置”。它的职责并入最后的“设置”页。

顶部工具栏只显示：

- 素材库名称。
- 健康状态。
- 可用原视频数量。
- 待处理数量。

顶部不显示完整本机路径。完整路径只在设置页和诊断详情中展示。

## 设置页

设置页是管理端的配置中心，放在左侧导航最后。

设置页分为四个区域：

### 素材来源

素材来源可以配置多个文件夹。

每个素材来源包含：

- 来源名称。
- 本机文件夹路径。
- 是否启用扫描。
- 最近扫描时间。
- 已发现原视频数量。
- 新增未处理数量。
- 路径状态。

操作：

- 添加素材来源。
- 移除素材来源。
- 启用或停用来源。
- 扫描选中来源。
- 扫描全部来源。

说明：

- `source-videos` 不再是唯一来源，只是默认素材来源。
- 目录可包含子文件夹。
- 原视频清单不得保存不可移植的绝对路径。
- 单视频清单应记录来源编号和来源内相对路径。

### 预处理产物库

预处理产物库用于保存：

- 视频清单。
- 文案。
- 字幕。
- 封面。
- 关键帧。
- 语音转文字中间产物。
- 搜索索引。
- 预处理日志。
- 健康诊断报告。

设计决策：

- 预处理产物库建议只有一个。
- 默认位置为公共素材库根目录下的 `.mixlab-library`。
- 高级模式允许选择产物库位置，但同一个公共素材库只能绑定一个产物库。
- 改变产物库位置属于迁移动作，需要单独确认和校验，不是普通设置项。

原因：

- 视频编号、索引、文案、可见性和日志必须统一管理。
- 多个产物库会造成搜索索引分裂、状态不一致和剪辑端可见性混乱。

### 运行策略

运行策略包含：

- 语音转文字服务状态。
- 当前模型。
- 音频模式。
- 并发任务数。
- 单视频失败后的处理策略。
- 是否自动扫描。
- 是否自动加入预处理队列。
- 是否自动发布索引。
- 视频处理组件状态。

操作：

- 保存运行策略。
- 测试语音转文字配置。
- 检查视频处理组件。

密钥处理：

- 页面只显示“已配置”或“未配置”。
- 不显示、不编辑、不导出密钥值。
- 编辑密钥仍由本地环境配置或桌面壳安全能力完成。

### 路径与权限校验

展示所有关键路径的中文校验结果：

- 素材来源是否可读。
- 预处理产物库是否可写。
- 视频清单是否可读写。
- 索引目录是否可读写。
- 日志目录是否可写。
- 磁盘空间是否充足。

每条校验展示：

- 检查对象。
- 检查目的。
- 当前结果。
- 失败影响。
- 处理建议。
- 技术详情。

## 仪表盘

仪表盘不只显示状态数量，还要表达生产能力、素材规模、剪辑使用情况和风险。

### 素材规模

- 原视频总数。
- 可用原视频数。
- 未处理原视频数。
- 队列中原视频数。
- 正在处理原视频数。
- 处理失败原视频数。
- 待发布索引原视频数。
- 原视频总大小。
- 原视频总时长。
- 已可用视频总时长。
- 未处理视频总时长。

### 文案与索引

- 已生成文案视频数。
- 文案总字数。
- 文案分段总数。
- 已进入搜索索引的视频数。
- 当前索引版本。
- 最近索引更新时间。
- 待发布索引数量。

### 预处理产能

- 今日处理完成数。
- 今日处理失败数。
- 最近一小时处理完成数。
- 平均每小时处理时长。
- 平均单小时可处理原视频时长。
- 平均单视频处理耗时。
- 队列预计完成时间。
- 失败最多的阶段。
- 重试成功数量。

### 剪辑端使用

需要新增使用事件统计，先支持从本地接口累计：

- 搜索请求次数。
- 有结果搜索次数。
- 无结果搜索次数。
- 原视频详情查看次数。
- 文案选段次数。
- 加入剪辑清单次数。
- 提交剪切任务次数。
- 剪切成功次数。
- 剪切失败次数。
- 生成本地素材数量。
- 最近使用的关键词。
- 被使用最多的原视频。

说明：

- 当前没有登录用户体系，所以先按工作台实例或本地工作区统计，不做个人绩效统计。
- 没有接入统计前，对应卡片显示“尚未开始记录”，不能显示假数据。

### 风险摘要

- 磁盘空间风险。
- 路径不可访问风险。
- 失败任务风险。
- 索引未发布风险。
- 语音转文字配置风险。
- 视频处理组件风险。

## 原视频管理

原视频管理列表仍负责资产管理，但需要更清楚地表达状态和入口。

列表字段：

- 视频编号。
- 封面。
- 标题。
- 文件名。
- 素材来源。
- 来源内相对路径。
- 时长。
- 文件大小。
- 预处理状态。
- 剪辑师是否可见。
- 标签。
- 最近更新时间。

操作：

- 扫描新增视频。
- 将未处理视频加入队列。
- 重试失败视频。
- 编辑公开元数据。

交互：

- 点击已可用原视频进入原视频详情页。
- 点击未处理、队列中、处理中、处理失败、待发布索引的视频，也可以进入详情页，但详情页要明确显示“剪辑师暂不可见”和原因。
- 封面必须显示真实图片；如果没有封面，显示中文占位图，不显示破图。

## 原视频详情页

新增原视频详情页，用于查看某个原视频完整预处理数据。

详情页展示：

### 基本信息

- 视频编号。
- 标题。
- 文件名。
- 素材来源。
- 来源内相对路径。
- 时长。
- 分辨率。
- 帧率。
- 编码格式。
- 文件大小。
- 内容指纹。
- 剪辑师可见状态。

### 预处理状态

- 当前状态。
- 当前阶段。
- 最近任务编号。
- 开始时间。
- 完成时间。
- 耗时。
- 失败阶段。
- 失败原因。
- 重试次数。
- 日志入口。

### 产物完整性

- 文案是否存在。
- 字幕是否存在。
- 封面是否存在。
- 关键帧是否存在。
- 搜索索引是否包含该视频。
- 当前索引版本。

### 文案数据

- 文案总字数。
- 分段数量。
- 平均分段时长。
- 完整文案预览。
- 字幕预览。

### 视觉数据

- 封面图。
- 关键帧列表。
- 可播放原视频预览。

### 公开元数据

- 标题。
- 标签。
- 说明。
- 讲师。
- 课程。
- 分类。

## 预处理队列

预处理队列页面必须先解释队列来源，再提供操作。

顶部数据：

- 未处理原视频数量。
- 未处理视频总时长。
- 未处理视频总大小。
- 已排队数量。
- 正在处理数量。
- 今日完成数量。
- 今日失败数量。

“加入预处理队列”按钮旁边必须显示影响范围：

- 将加入多少个未处理视频。
- 预计总时长。
- 来自哪些素材来源。

列表分组：

- 未处理，等待加入队列。
- 队列中，等待后台处理。
- 正在处理。
- 待发布索引。
- 已完成。
- 处理失败。

每行展示：

- 视频编号。
- 标题或文件名。
- 素材来源。
- 时长。
- 文件大小。
- 当前阶段。
- 进度。
- 耗时。
- 失败原因。
- 可执行操作。

## 健康诊断

健康诊断页面必须中文解释每个检查项。

每个检查项展示：

- 检查名称。
- 检查目的。
- 当前结果。
- 失败影响。
- 建议处理方式。
- 技术详情。

检查项中文名称：

- 公共素材库根目录。
- 素材来源可读性。
- 预处理产物库可写性。
- 素材数量一致性。
- 原视频清单完整性。
- 当前搜索索引。
- 视频处理组件。
- 视频信息读取组件。
- 语音转文字配置。
- 本地素材清单。

健康诊断不再展示英文标签作为主要信息。

## 索引与发布

页面名称调整为“索引与发布”。

展示：

- 当前索引版本。
- 已索引视频数。
- 待发布索引视频数。
- 最近发布时间。
- 索引校验结果。
- 历史索引版本。

操作：

- 发布待索引视频。
- 校验当前索引。
- 查看索引技术详情。

## 数据与接口需求

新增或调整接口：

- 获取设置配置。
- 保存素材来源配置。
- 添加素材来源。
- 移除素材来源。
- 扫描指定素材来源。
- 获取管理端指标摘要。
- 获取原视频详情。
- 获取原视频封面绝对地址。
- 获取原视频文案数据。
- 获取原视频关键帧数据。
- 获取健康诊断中文解释。
- 记录剪辑端使用事件。
- 查询剪辑端使用统计。

前端媒体地址规则：

- 接口返回的相对媒体路径必须由前端解析到管理端接口地址。
- 破图必须降级为中文占位封面。

## 验收标准

1. 管理端所有可见页面没有英文界面文案。
2. 左侧导航最后一项是“设置”，不存在“公共素材库设置”独立页面。
3. 顶部不再显示完整本机路径。
4. 设置页可以表达多个素材来源和一个预处理产物库。
5. 仪表盘展示素材规模、文案索引、预处理产能、剪辑端使用和风险摘要。
6. 原视频列表封面正常显示；无封面时显示中文占位，不显示破图。
7. 点击原视频可进入原视频详情页。
8. 原视频详情页能展示预处理状态、产物完整性、文案统计、视觉数据和公开元数据。
9. 预处理队列能明确展示未处理视频数量、总时长、总大小和加入队列影响范围。
10. 健康诊断检查项有中文名称、检查目的、影响和处理建议。

## 不在本阶段解决

- 不引入登录和个人权限体系。
- 不做剪辑师端页面重构。
- 不在浏览器中直接编辑密钥明文。
- 不在浏览器中直接启动长期后台工作进程。
- 不做多产物库合并或迁移工具。
