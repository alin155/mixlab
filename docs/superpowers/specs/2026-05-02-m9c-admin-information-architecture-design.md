# M9C 管理端信息架构修正设计

## 背景

M9B 已经让管理端控件接入真实接口，但页面信息架构暴露出几个产品问题：

- 顶部路径展示没有可操作意义。
- 公共素材库设置页像只读展示，不像真正的配置中心。
- 设置页和公共素材库设置页职责重叠。
- 原视频封面接口返回的是相对地址，前端没有解析到管理端接口地址。
- 预处理队列缺少“哪些视频未处理、数量多少、操作会影响什么”的上下文。
- 健康诊断仍是工程报告语言，管理员难以判断检查目的和处理建议。
- 原视频列表缺少进入预处理详情的路径。
- 剪辑端缺少登录和准入机制，管理端无法管理剪辑师用户，也无法把搜索、选段、剪切等使用数据绑定到具体用户。

本阶段目标是修正管理端的产品结构和可理解性，再继续扩展功能。

## 总体原则

1. 管理端页面不出现英文界面文案。
   技术名词必须转成中文展示，例如：
   - `ready` 显示为“已可用”。
   - `queued` 显示为“队列中”。
   - `processing` 显示为“处理中”。
   - `unprocessed` 显示为“未处理”。
   - `failed` 显示为“处理失败”。
   - `index-required` 显示为“待发布索引”。
   - `Doctor` 显示为“健康诊断”。
   - `ASR` 显示为“语音转文字”。
   - `API Key` 显示为“接口密钥”。
2. 页面名称、按钮、表头、状态、阶段、提示、错误说明都必须中文化。
3. 技术详情可以保留真实路径、文件名和协议字段，但默认放在“技术详情”折叠区或详情页。
4. 管理端围绕“公共素材从哪里来、预处理产物存哪里、现在生产到哪一步、剪辑师能用多少”组织。
5. 浏览器页面不直接保存密钥，不直接启动长期后台工作进程；这些仍由本地服务或桌面壳承接。
6. 剪辑端必须先登录才能使用公共素材库。登录采用“输入用户名申请、管理端审核通过、下次自动登录”的轻量准入模式。
7. 剪辑端使用事件必须绑定到已审核用户，用于管理端统计使用情况。

## 导航结构

左侧导航调整为：

1. 仪表盘
2. 原视频管理
3. 预处理队列
4. 索引与发布
5. 健康诊断
6. 剪辑师用户
7. 设置

删除独立的“公共素材库设置”。它的职责并入最后的“设置”页。

新增“剪辑师用户”页面，用于审核登录申请、管理剪辑端用户和查看用户维度使用数据。

顶部工具栏只显示：

- 素材库名称。
- 健康状态。
- 可用原视频数量。
- 待处理数量。

顶部不显示完整本机路径。完整路径只在设置页和诊断详情中展示。

## 设置页

设置页是管理端的配置中心，放在左侧导航最后。

设置页分为四个区域：

### 素材来源

素材来源可以配置多个文件夹。

每个素材来源包含：

- 来源名称。
- 本机文件夹路径。
- 是否启用扫描。
- 最近扫描时间。
- 已发现原视频数量。
- 新增未处理数量。
- 路径状态。

操作：

- 添加素材来源。
- 移除素材来源。
- 启用或停用来源。
- 扫描选中来源。
- 扫描全部来源。

说明：

- `source-videos` 不再是唯一来源，只是默认素材来源。
- 目录可包含子文件夹。
- 原视频清单不得保存不可移植的绝对路径。
- 单视频清单应记录来源编号和来源内相对路径。

### 预处理产物库

预处理产物库用于保存：

- 视频清单。
- 文案。
- 字幕。
- 封面。
- 关键帧。
- 语音转文字中间产物。
- 搜索索引。
- 预处理日志。
- 健康诊断报告。

设计决策：

- 预处理产物库建议只有一个。
- 默认位置为公共素材库根目录下的 `.mixlab-library`。
- 高级模式允许选择产物库位置，但同一个公共素材库只能绑定一个产物库。
- 改变产物库位置属于迁移动作，需要单独确认和校验，不是普通设置项。

原因：

- 视频编号、索引、文案、可见性和日志必须统一管理。
- 多个产物库会造成搜索索引分裂、状态不一致和剪辑端可见性混乱。

### 运行策略

运行策略包含：

- 语音转文字服务状态。
- 当前模型。
- 音频模式。
- 并发任务数。
- 单视频失败后的处理策略。
- 是否自动扫描。
- 是否自动加入预处理队列。
- 是否自动发布索引。
- 视频处理组件状态。

操作：

- 保存运行策略。
- 测试语音转文字配置。
- 检查视频处理组件。

密钥处理：

- 页面只显示“已配置”或“未配置”。
- 不显示、不编辑、不导出密钥值。
- 编辑密钥仍由本地环境配置或桌面壳安全能力完成。

### 路径与权限校验

展示所有关键路径的中文校验结果：

- 素材来源是否可读。
- 预处理产物库是否可写。
- 视频清单是否可读写。
- 索引目录是否可读写。
- 日志目录是否可写。
- 磁盘空间是否充足。

每条校验展示：

- 检查对象。
- 检查目的。
- 当前结果。
- 失败影响。
- 处理建议。
- 技术详情。

- 加入剪辑清单次数。
- 提交剪切任务次数。
- 剪切成功次数。
- 剪切失败次数。
- 生成本地素材数量。

- 各剪辑师文案选段次数。

- 各剪辑师生成本地素材数量。

- 用户维度统计用于理解素材库使用情况和排查问题，不作为复杂权限体系或绩效系统。
- 没有接入统计前，对应卡片显示“尚未开始记录”，不能显示假数据。

## 剪辑师用户

新增“剪辑师用户”页面，放在“健康诊断”和“设置”之间。

### 登录申请流程

剪辑端首次打开时：

1. 显示登录申请页。
2. 剪辑师输入用户名。
3. 剪辑端生成本机设备编号。
4. 剪辑端提交登录申请。
5. 申请状态为“待审核”。
6. 管理端在“剪辑师用户”页面看到申请。
7. 管理员审核通过后，该用户可以登录。
8. 剪辑端保存本地登录凭证，下次自动登录。

如果申请被拒绝：

- 剪辑端显示“申请未通过”。
- 可以重新提交申请。

如果用户被停用：

- 剪辑端下一次刷新或访问接口时退出登录。
- 页面显示“账号已停用，请联系管理员”。

### 用户状态

用户状态包含：

- 待审核。
- 已通过。
- 已拒绝。
- 已停用。

### 用户字段

每个剪辑师用户记录：

- 用户编号。
- 用户名。
- 显示名称。
- 状态。
- 首次申请时间。
- 审核通过时间。
- 最近登录时间。
- 最近使用时间。
- 设备数量。
- 备注。

每个设备记录：

- 设备编号。
- 设备名称。
- 首次申请时间。
- 最近登录时间。
- 状态。

说明：

- 用户名由剪辑师输入，管理端审核后生效。
- 同一用户名允许绑定多个设备，但管理端必须能看到设备列表。
- 后续可以扩展为手机号、企业账号或密码登录；本阶段不引入复杂账号体系。

### 管理端操作

管理端可执行：

- 通过申请。
- 拒绝申请。
- 停用用户。
- 恢复用户。
- 删除本机设备授权。
- 修改显示名称。
- 添加备注。
- 查看用户使用统计。

### 剪辑端登录体验

剪辑端未登录时不可进入主工作台。

剪辑端登录状态：

- 未申请：显示用户名输入和提交申请按钮。
- 待审核：显示等待审核状态和申请时间。
- 已通过：自动进入工作台。
- 已拒绝：显示拒绝状态和重新申请入口。
- 已停用：显示停用提示。

剪辑端顶部应显示当前用户名，但不展示复杂账号菜单。

### 本地凭证

剪辑端通过本地持久化凭证实现下次自动登录。

凭证包含：

- 用户编号。
- 设备编号。
- 本地会话令牌。

安全边界：

- 会话令牌不得写入公共素材库公开清单。
- 会话令牌不得进入诊断报告或日志明文。
- 管理端可以撤销用户或设备授权。

本模式是局域网和本地部署场景下的轻量准入，不替代互联网级账号密码安全体系。

### 用户维度统计

所有剪辑端使用事件都应带上用户编号和设备编号。

事件类型包括：

- 搜索。
- 查看原视频。
- 查看完整文案。
- 选择文案片段。
- 加入剪辑清单。
- 提交剪切任务。
- 剪切成功。
- 剪切失败。
- 生成本地素材。
- 复用本地素材。

每条事件记录：

- 事件编号。
- 用户编号。
- 设备编号。
- 事件类型。
- 发生时间。
- 关联原视频编号。
- 关联任务编号。
- 搜索关键词。
- 选段时长。
- 结果状态。

统计口径：

- 总量统计进入仪表盘。
- 用户排行进入“剪辑师用户”页面。
- 单用户详情展示该用户最近使用记录和常用素材。

### 风险摘要

- 磁盘空间风险。
- 路径不可访问风险。
- 失败任务风险。
- 索引未发布风险。
- 语音转文字配置风险。
- 视频处理组件风险。

- 素材来源。

- 文件大小。
- 预处理状态。

## 验收标准

1. 管理端所有可见页面没有英文界面文案。
2. 左侧导航最后一项是“设置”，不存在“公共素材库设置”独立页面。
