# M14 剪辑端真实搜索与文案阅读器设计

## 目标

M14 完成剪辑师工作台的第二条真实主路径：从已发布公共素材中搜索，进入原视频详情阅读完整文案，连续选择文案片段，并加入待剪清单。

## 范围

- 搜索页使用 Cutter API 的真实搜索结果。
- 搜索结果按原视频聚合展示，不做句子瀑布流。
- 搜索结果可跳转到原视频详情，并携带搜索词和命中文案段落。
- 原视频详情页高亮命中文案段落。
- 文案阅读器支持连续选段，选段只生成一个待剪片段。
- 加入待剪清单后进入待剪清单页面。

## 非目标

- 不在 M14 执行真实 FFmpeg 剪切。
- 不做复杂素材推荐、语义扩展、同义词管理。
- 不做多用户协同待剪清单。
- 不做本地素材库复用闭环，这进入 M15。

## 设计决策

### 路由与上下文

剪辑端使用 hash 保存搜索和文案阅读上下文：

- `#search?query=现金流`
- `#source-detail/V000001?query=现金流&segments=V000001-S000001,V000001-S000002`

这样刷新页面不会丢失搜索词、当前原视频和命中文案定位。

### 搜索页

搜索页包含一个可提交的搜索框。提交后更新 hash，并由应用层调用 Cutter API 搜索。结果按原视频分组，每组展示封面、标题、命中数量、最佳摘要和少量上下文文案。点击“查看完整文案”进入对应原视频详情。

### 文案阅读器

详情页展示视频播放器、完整文案、分段文案列表和连续选择面板。搜索命中的段落用高亮样式标记。剪辑师点击分段按钮选择连续范围，第一次点击设置起点，第二次点击设置终点，随后再次点击重新开始选择。

### 待剪清单

加入待剪清单沿用现有 `CutListItem` 模型，来源视频、起止句、时间范围、选中文案、剪切模式都写入本地待剪清单。M14 不提交剪切任务，只准备 M15 的真实剪切输入。

## 验收标准

1. 搜索页提交搜索词后调用真实搜索 API。
2. 搜索结果按原视频聚合，并能跳转到对应详情页。
3. 详情页能根据 hash 高亮命中文案段落。
4. 连续选择文案段落后，加入待剪清单生成一个片段。
5. 待剪清单展示来源、时间范围、选中文案和剪切模式。
6. 原有公共素材库、登录、剪切队列页面不回归。
