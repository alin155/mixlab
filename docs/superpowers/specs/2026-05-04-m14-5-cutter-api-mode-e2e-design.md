# M14.5 剪辑端 API 模式端到端联调设计

## 背景

M13 已经让剪辑端接入真实 Cutter API：登录审核、公共素材库、原视频详情、搜索和媒体地址都走 API。M14.1 到 M14.4 把剪辑端产品主路径收敛为“搜、选、剪、复用”，并让真实 API 模式下的剪切任务提交后自动调用本机剪切流水线。

当前还缺少一层生产化保障：开发者和管理员很难一眼确认剪辑端当前是不是连到了真实 Cutter API、是否有本地剪切工作区、FFmpeg 是否能完成真实视频剪切、已剪片段是否真的进入本地素材库。M14.5 的目标是把这条真实链路做成可诊断、可重复验证的端到端闭环。

## 产品目标

剪辑师和管理员不需要理解 API 端口、会话头、工作区目录、FFmpeg 命令这些底层细节，但系统必须能清楚回答四个问题：

1. 剪辑端是否连接到真实 Cutter API，而不是 fixture。
2. 当前登录身份是否是管理端审核通过的剪辑师。
3. 本地剪切工作区是否启用，剪切结果会写入哪里。
4. 从公共素材搜索到本机剪切再到本地素材复用，真实链路是否跑通。

## 范围

本轮做：

- Cutter API 新增只读运行状态接口，用中文友好的字段描述真实运行模式、素材库、工作区、FFmpeg 和本地素材数量。
- 剪辑端 API client 接入运行状态接口。
- 剪辑端设置页展示“真实模式联调状态”，让剪辑师能看到 API 地址、登录用户、本地剪切工作区状态和本地素材数量。
- 新增可重复端到端烟测脚本：自动创建临时真实素材库、生成可播放 MP4、写入 ready 文案和发布状态、创建审核通过的剪辑师会话、启动 Cutter API、启动剪辑端 Vite API 模式、通过浏览器完成搜索、选文案、剪切、等待本地素材回流、验证本地素材优先。
- 烟测使用真实 FFmpeg 生成源视频，并使用 Cutter API 默认 FFmpeg 剪切执行器，不使用假的 cut runner。
- 所有新增用户可见文案使用中文。

本轮不做：

- 不做 Tauri 打包。
- 不做后台常驻剪切服务。
- 不做公共素材下载同步到本地。
- 不做管理员一键启动剪辑端和 API 的图形化启动器。
- 不做多用户并发剪辑压力测试。

## 推荐方案

采用“运行状态接口 + 浏览器端到端烟测”的方案。

Cutter API 是真实运行模式的唯一可信边界。它知道公共素材库根目录、本地工作区是否启用、FFmpeg 是否可用、ready 素材数量、本地导出素材数量。新增 `/cutter/runtime-status` 让剪辑端设置页可以展示真实联调状态，也让烟测脚本可以在浏览器操作前确认服务已经准备好。

端到端烟测不依赖用户现有 35T 素材库，避免破坏真实数据。脚本创建临时素材库和临时工作区，生成一段极短的真实 MP4，按协议写入 ready 素材和文案，审核一个测试剪辑师，然后用真实 Cutter API 和真实剪辑端页面跑完整主路径。这样既不会污染用户素材库，又能证明产品链路是真实的。

## 数据流

### 运行状态

1. 剪辑端 API 模式启动。
2. `CutterApp` 使用当前审核通过的设备会话调用 `GET /cutter/runtime-status`。
3. Cutter API 校验会话。
4. Cutter API 返回：
   - `mode_label`：真实 Cutter API 模式。
   - `api_ready`：服务是否可用。
   - `library_root_label`：公共素材库名称或脱敏路径标签。
   - `available_video_count`：剪辑端可见 ready 素材数。
   - `workspace_enabled`：是否启用本地剪切工作区。
   - `workspace_root_label`：本地工作区名称或脱敏路径标签。
   - `local_clip_count`：当前本地可复剪素材数。
   - `ffmpeg_status`：可用、不可用。
   - `ffmpeg_source`：内置或环境配置。
   - `current_user`：当前审核通过的剪辑师用户名。
5. 设置页展示为中文只读状态，不暴露会话令牌。

### 端到端烟测

1. 生成临时公共素材库和本地工作区。
2. 使用 bundled FFmpeg 生成真实 MP4。
3. 扫描源视频并写入 transcript、SRT、keyframes、cover。
4. 发布为 cutter-visible ready。
5. 审核测试剪辑师并拿到会话。
6. 启动 Cutter API，传入 `library_root` 和 `workspace_root`。
7. 启动 Cutter Web，注入 `VITE_MIXLAB_CUTTER_API_BASE_URL`。
8. Playwright 注入剪辑师会话到浏览器 localStorage。
9. 打开 `#material-locator?query=现金流`。
10. 点击文案片段并执行“剪切这段”。
11. 等待“剪切完成 · 本地素材已更新 1”。
12. 打开本地素材库确认新增本地素材。
13. 回到素材定位页确认本地素材分组仍排在公共原素材之前。
14. 脚本停止临时服务并保留失败时的错误信息。

## 错误处理

- `runtime-status` 会话无效：返回已有登录错误，剪辑端回到登录门。
- 没有工作区：运行状态显示“未启用本地剪切工作区”，剪切相关接口仍返回中文可读错误。
- FFmpeg 不可用：运行状态显示“不可用”，烟测直接失败并输出原因。
- 端到端烟测失败：脚本输出当前阶段、API URL、临时库路径和工作区路径，方便复现。
- 浏览器未看到预期文案或本地素材：脚本失败，不误报成功。

## 验收标准

1. API 模式设置页能看到“真实模式联调状态”。
2. 设置页能显示当前 API 地址、审核用户、ready 可用素材数、本地工作区状态、FFmpeg 状态、本地素材数。
3. `/cutter/runtime-status` 需要审核通过的剪辑师会话，不能匿名读取。
4. `npm run smoke:cutter-api-web` 能在临时素材库中跑通真实 MP4 播放、文案选择、真实 FFmpeg 剪切、本地素材回流和本地素材优先搜索。
5. 不向公共素材库写入本地剪切结果；剪切结果只进入 `MIXLAB_CUTTER_WORKSPACE_ROOT`。
6. 所有新增用户可见文案为中文。
7. `npm test`、`npm run typecheck`、`npm run build:cutter-web`、`npm run smoke:cutter-api-web` 通过。
