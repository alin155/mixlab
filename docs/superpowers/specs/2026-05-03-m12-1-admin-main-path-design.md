# M12.1 管理端主路径重构设计

日期：2026-05-03
范围：素材库管理端信息架构、预处理流水线、索引发布归位、设置页职责收敛

## 1. 背景

当前管理端已经具备扫描、入队、预处理、索引发布、健康诊断、原视频元数据管理等能力，但功能入口被拆散在多个页面。管理员在真实使用时并不想理解“扫描、入队、启动服务、发布索引”这些工程步骤，而是想完成一个业务目标：

> 把新增或存量原视频持续变成剪辑师可搜索、可查看、可复用的公共素材。

因此 M12.1 的目标不是增加更多按钮，而是把管理端收敛成清晰的主路径：显式智能扫描、一键持续预处理流水线、自动增量发布、页面按钮收敛。

## 2. 用户核心任务

管理端必须优先服务四个任务：

1. 第一次初始化素材库，并预处理已有原视频。
2. 新增源素材后，显式触发一次智能扫描并继续预处理。
3. 预处理完成一个视频，就自动发布一个视频，使剪辑端尽快可见。
4. 长时间处理 35T 素材时，管理员可以掌握进度、风险、失败原因和预计完成时间。

管理端不应要求管理员日常手动执行扫描、入队、发布索引这些拆碎动作。

## 3. 信息架构调整

### 3.1 保留页面

管理端日常导航收敛为六个页面：

- `仪表盘`：全局状态、智能扫描、下一步建议、启动或继续预处理流水线。
- `原视频管理`：查看素材资产台账、公开元数据、单个原视频详情。
- `预处理`：生产监控页，承载原“预处理队列”和“索引与发布”的日常信息。
- `健康诊断`：定位系统问题并给出处理建议。
- `剪辑师用户`：审核剪辑师登录申请、停用用户、查看使用统计。
- `设置`：配置素材来源、产物库、音频模式、并发数、语音识别配置。

### 3.2 移除独立页面

`索引与发布`不再作为日常独立页面展示。

原因：

- 索引发布是预处理流水线的一部分，不应成为管理员日常必经步骤。
- 单个视频预处理成功后应自动增量发布。
- 异常索引问题应在`预处理`和`健康诊断`中显示，而不是让管理员主动进入单独页面排查。

兼容要求：

- 旧路由 `#/index-publish` 可保留为兼容入口，但应跳转或渲染到`预处理`页面的“索引状态”区域。
- 新导航中不展示`索引与发布`。

## 4. 页面职责与按钮边界

### 4.1 仪表盘

仪表盘是唯一主控台。

保留或新增：

- `智能扫描`
- `启动预处理流水线`
- `暂停预处理流水线`
- `继续预处理流水线`
- `重试失败视频`，仅当存在失败视频时显示为建议动作

智能扫描结果应展示：

- 新发现原视频数量
- 未处理数量
- 队列中数量
- 正在处理数量
- 待自动发布数量
- 失败数量
- 预计总时长和预计完成时间
- 磁盘、语音识别、FFmpeg、素材路径风险
- 推荐主动作

### 4.2 原视频管理

原视频管理只负责资产台账和元数据，不放全局生产动作。

保留：

- 搜索和筛选
- 查看详情
- 保存公开说明
- 单个失败视频重试或单个视频重新处理，可作为行级高级操作

移除：

- 扫描新增视频
- 加入预处理队列
- 处理全部未处理
- 查看发布清单

### 4.3 预处理

`预处理队列`改名为`预处理`。

职责：

- 展示持续预处理流水线是否运行。
- 展示当前正在处理的视频、阶段、耗时、预计剩余。
- 展示等待队列、完成记录、失败记录。
- 展示自动增量发布状态。
- 展示索引版本、当前索引可用数量、待发布或发布失败原因。

保留：

- `启动预处理流水线`
- `暂停预处理流水线`
- `继续预处理流水线`
- `重试失败视频`

移除：

- `加入预处理队列`作为页面主按钮。

### 4.4 设置

设置页只负责配置，不放生产动作。

保留：

- 素材库名称
- 素材来源文件夹
- 预处理产物库路径
- 音频模式
- 并发任务数
- 语音识别配置测试
- 保存设置

移除：

- 初始化素材库
- 扫描源视频
- 自动扫描
- 自动入队
- 自动发布索引

说明：

- 初始化素材库应由智能扫描或首次启动流水线自动判断并执行。
- 扫描源视频应由智能扫描显式触发。
- 入队和发布索引是流水线内部动作，不作为设置开关暴露。

## 5. 预处理流水线语义

### 5.1 流水线启动

管理员点击`启动预处理流水线`后，系统执行：

```text
确认素材库结构
-> 扫描启用的素材来源
-> 发现新增原视频并标记为 unprocessed
-> 将 unprocessed 加入 queued
-> 启动持续预处理服务
-> 按并发任务数领取 queued 视频
-> 单个视频处理成功后进入 index-required
-> 自动补齐封面、关键帧和索引
-> 成功后进入 ready
-> 刷新统计与可搜索索引
```

### 5.2 持续运行

`并发任务数`只表示同时处理多少个视频，不表示本次只处理多少个视频。

流水线默认持续运行，直到：

- 队列清空；
- 管理员点击暂停；
- 健康诊断发现阻塞问题；
- 系统遇到不可恢复的配置错误。

单个视频失败不得停止整条流水线。

### 5.3 新增素材

新增素材推荐流程：

```text
管理员把新视频放入素材来源文件夹
-> 回到仪表盘点击智能扫描
-> 系统列出新增数量和处理建议
-> 管理员点击启动或继续预处理流水线
-> 新视频自动进入队列并持续处理
```

不做后台自动扫描。

原因：

- 避免拷贝未完成的视频被误扫描。
- 避免移动硬盘或网络盘短暂断开时产生误报。
- 管理员明确知道“我刚添加了素材，现在开始处理”。

## 6. 自动增量发布

自动发布索引是默认产品规则，不再作为设置开关。

规则：

- 单个视频文案、字幕、封面、关键帧和索引全部生成成功后，自动进入 `ready`。
- `ready` 视频立即进入当前可搜索索引。
- 剪辑端刷新后可见。
- 发布失败的视频停留在 `index-required` 或 `failed`，不得进入剪辑端搜索。
- 失败原因在`预处理`页的索引状态区域和`健康诊断`中展示。

## 7. 状态模型

原视频状态继续使用现有枚举：

```text
unprocessed
queued
processing
index-required
ready
failed
```

流水线状态新增或明确为：

```text
stopped     未运行
scanning    扫描素材
running     持续处理
pausing     请求暂停
paused      已暂停
draining    队列收尾
failed      流水线异常
```

页面文案必须全部中文展示。

## 8. 失败处理

失败分为两类：

- 单视频失败：记录失败阶段和原因，进入失败列表，流水线继续。
- 系统阻塞：例如素材路径不可访问、ASR Key 缺失、FFmpeg 不可用、磁盘空间不足，流水线暂停并要求管理员处理。

失败视频操作：

- `重试失败视频`：把失败视频重新加入队列。
- 单条失败视频可以在原视频详情中查看完整预处理数据。

## 9. 验收标准

M12.1 通过标准：

1. 导航中不再显示`索引与发布`。
2. `预处理队列`改名为`预处理`。
3. 设置页不再显示初始化、扫描、自动扫描、自动入队、自动发布索引。
4. 原视频管理页不再显示全局扫描和全局入队按钮。
5. 仪表盘智能扫描能生成可理解的处理建议。
6. 点击启动流水线后，系统能自动扫描、入队、处理、发布索引。
7. 预处理页能展示当前处理、队列、完成、失败、索引状态和预计时间。
8. 单个视频完成后自动进入剪辑端可搜索范围。
9. 单个视频失败不阻塞后续视频。
10. 全部用户可见文案为中文，不暴露内部 stage、日志路径或英文错误。

## 10. 不做事项

M12.1 不做：

- 后台定时自动扫描。
- 多机器分布式预处理。
- NAS 专项权限策略。
- 剪辑端 UI 重构。
- 复杂调度策略，例如优先级队列、暂停单个视频、跳过单个视频。

这些事项可以在后续阶段单独设计。
