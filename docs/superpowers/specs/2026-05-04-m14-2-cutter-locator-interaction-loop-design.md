# M14.2 剪辑端素材定位真实交互闭环设计

## 目标

M14.2 在 M14.1 的主工作台基础上，把“素材定位”从静态工作台推进为可真实操作的搜、选、预览、剪切闭环。

本轮不新增一级页面，不改变已经确认的五个一级导航：

```text
素材定位 / 剪切任务 / 本地素材 / 公共素材库 / 设置
```

核心目标：

```text
搜索命中 -> 打开视频 -> 点击文案定位画面 -> 拖选文案形成选区 -> 播放预览 -> 剪切这段 -> 当前页看到任务反馈
```

## 产品原则

- 视频仍是主视觉，文案仍是主操作系统。
- 剪辑师不需要看到时间码、segment、队列 id 或内部状态词。
- 点击文案是定位画面，不是加入片段。
- 拖选文案是形成剪切选区，不需要额外“设为起点/终点”模式。
- “播放预览”必须围绕当前选区工作。
- “剪切这段”必须留在当前素材定位页，并给出轻量反馈。

## 功能范围

### 1. 点击文案定位播放器

点击自然文案中的一句时：

- 当前视频跳到该句 `begin_ms` 前约 0.5 秒。
- 如果 `begin_ms` 小于 0.5 秒，则从 0 秒开始。
- 当前句成为单句选区。
- 播放器不自动开始播放，避免误打断剪辑师判断。

### 2. 拖选文案生成连续选区

拖选文案时：

- 鼠标按下所在 span 作为起点。
- 鼠标抬起所在 span 作为终点。
- 系统按文案顺序生成连续选区。
- 反向拖选也能得到正确的连续区间。
- 拖选完成后不能再被浏览器随后的 click 事件覆盖成单句选区。

### 3. 播放当前选区

点击“播放预览”时：

- 视频跳到选区起点前约 0.5 秒。
- 视频开始播放。
- 播放到选区终点时自动暂停。
- 如果浏览器阻止自动播放，页面不报错，保留当前选区。

### 4. 搜索结果打开和命中高亮

点击素材定位搜索结果时：

- 当前素材定位页内打开对应视频。
- 不跳转到原视频详情子页面。
- 文案区高亮该搜索结果的命中句。
- 默认选中该搜索结果覆盖的连续范围。
- 公共原素材和本地素材使用同一套交互模式。

### 5. 剪切任务轻反馈

点击“剪切这段”后：

- 立即创建后台剪切任务。
- 当前页面不跳转。
- 选区清空。
- 素材定位页显示轻提示，例如：

```text
已加入剪切任务 · 等待中 1
```

- 右侧任务摘要同步显示等待、剪切中、完成数量。

## 非目标

M14.2 不做：

- 本地视频导入。
- 公共素材下载或同步。
- 真实剪切 worker 的高级进度展示。
- 多选区批量剪切。
- 编辑选区起止时间的精细控件。
- 新建项目、合集或混剪工程管理。

## 验收标准

1. 点击文案句子会把播放器定位到该句起点附近。
2. 点击文案后形成单句选区。
3. 拖选多句文案会形成连续选区。
4. 反向拖选也能形成正确连续选区。
5. 拖选完成后不会被 click 事件覆盖成单句。
6. “播放预览”会从选区起点附近开始播放。
7. 视频播放到选区终点时自动暂停。
8. 点击搜索结果会在素材定位页内打开对应视频，不跳页。
9. 搜索命中句会高亮。
10. 搜索命中范围会默认成为当前选区。
11. 点击“剪切这段”后停留在素材定位页。
12. 点击“剪切这段”后页面显示任务轻反馈。
13. 点击“剪切这段”后任务摘要数量更新。
14. 页面不出现“片段篮”或“待剪清单”作为主流程概念。
15. 所有新增交互有自动化测试覆盖，并通过浏览器烟测。
