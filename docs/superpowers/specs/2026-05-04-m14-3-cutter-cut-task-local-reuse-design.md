# M14.3 剪辑端剪切任务与本地复用闭环设计

## 背景

M14.1 已经把剪辑端重构为“素材定位、剪切任务、本地素材、公共素材库、设置”五个一级入口。M14.2 已经完成素材定位页的真实搜、选、预览、剪切入口：剪辑师可以点击文案定位视频、拖选文案、播放预览，并把选区直接加入剪切任务。

M14.3 的目标是补完下一段闭环：剪切任务不再只是静态列表，而是能持续反映后台执行状态；剪切完成后的片段要进入本地素材库，并重新参与素材定位搜索。这样剪辑师的主流程变成：

`搜索文案 -> 进入视频 -> 拖选文案 -> 剪切这段 -> 任务完成 -> 本地素材复用`

## 产品目标

1. 剪辑师点击“剪切这段”后继续留在素材定位页，可以继续找素材，不被任务页打断。
2. 素材定位页要持续知道后台剪切任务状态，提示等待、剪切中、完成数量。
3. 剪切任务页要自动刷新，不要求剪辑师手动点“刷新任务”才能知道任务有没有完成。
4. 任务完成后，本地素材库自动增加完成片段。
5. 新增本地素材必须参与素材定位搜索，并且结果分组仍然先显示本地素材，再显示公共原素材。
6. 页面文案保持中文，不暴露 API 状态枚举、内部 job id、clip id 或底层路径作为主要体验信息。

## 非目标

M14.3 不做批量删除、批量暂停、任务优先级排序、任务拖拽排序、复杂失败诊断面板、下载同步、完整素材详情重构，也不改变 M14.1 的五个一级入口。

## 信息架构

### 素材定位

素材定位仍然是剪辑师首页。直接剪切成功后，页面顶部出现轻提示：

- 已加入剪切任务
- 当前等待、剪切中、已完成数量
- 已完成时提示“本地素材已更新”

提示不阻塞继续搜索和选段。剪辑师想看详细状态时，可以进入“剪切任务”。

### 剪切任务

剪切任务页承担生产可见性，不承担选段决策。页面应展示：

- 等待中数量
- 剪切中数量
- 已完成数量
- 失败数量
- 每个任务的中文状态、进度、选中文案、来源视频、时间范围

当存在 API handler 时，页面自动刷新任务状态。刷新节奏保持轻量，避免频繁请求影响剪辑端体验。

### 本地素材

本地素材页展示本机已生成的可复剪片段。剪切任务完成后，本地素材列表自动刷新；素材定位搜索也使用最新本地素材。剪辑师可以在素材定位中像处理公共原素材一样处理本地素材：进入视频、拖选文案、剪切这段。

## 数据流

1. 素材定位页调用 `createClipList` 和 `submitCutJobs` 创建剪切任务。
2. `CutterApp` 把提交成功的任务写入剪切任务状态。
3. API 模式下，`CutterApp` 周期性调用 `listCutJobs` 获取任务状态。
4. 当任务列表中出现新的完成任务，`CutterApp` 调用 `listLocalClips` 刷新本地素材。
5. `MaterialLocatorPage` 使用刷新的 `localClips` 重新生成本地搜索结果。
6. fixture 模式下，通过本地状态模拟“等待中 -> 剪切中 -> 已完成 -> 本地素材增加”，用于开发和浏览器烟测。

## 状态策略

新增轻量 helper 负责判断是否需要刷新本地素材：

- 统计上一轮和当前轮已完成任务数。
- 如果当前已完成任务数增加，则刷新本地素材。
- 如果 API 返回的任务带有完成状态但本地素材数未变化，页面不报错，只保留任务完成状态，等待下一轮刷新。

剪切任务自动刷新只在以下情况启用：

- API 模式且登录通过。
- 当前已有工作台数据。
- 当前页面是“素材定位”或“剪切任务”。
- 最近创建过剪切任务，或任务列表中存在等待中、剪切中任务。

这样可以保证主工作台状态及时，又避免所有页面无意义轮询。

## 错误处理

- 创建剪切任务失败：保留当前选区并显示中文错误。
- 刷新剪切任务失败：保留旧任务列表并显示中文错误。
- 刷新本地素材失败：不影响剪切任务显示，提示“本地素材刷新失败”。
- 单个任务失败：在剪切任务页以中文失败状态显示，保留重试入口。

## 测试策略

1. 状态 helper 单元测试：判断哪些路由需要自动刷新、何时需要刷新本地素材、任务计数是否正确。
2. 页面渲染测试：剪切任务页展示中文统计、自动刷新提示、本地素材页展示新增片段。
3. 应用状态测试：直接剪切后留在素材定位页；任务完成后刷新本地素材；本地素材参与搜索结果。
4. 浏览器烟测：在 fixture 模式下完成一次“剪切这段 -> 任务完成 -> 本地素材出现 -> 素材定位搜到本地结果”的链路。

## 验收标准

1. 点击“剪切这段”后仍停留在素材定位页。
2. 剪切任务状态可以自动变化，不依赖手动刷新。
3. 完成任务会让本地素材数量增加。
4. 本地素材结果在素材定位搜索中排在公共原素材前面。
5. 所有新增可见文案为中文。
6. `npm test`、`npm run typecheck`、`npm run build:cutter-web` 通过。
