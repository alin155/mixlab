# M16.4 剪辑端主题系统与本地素材组织设计

## 背景

M16.3 已经完成剪辑端剪切产出收口：剪切命名、交付目录、本地素材预处理、任务状态、项目删除和本地素材详情都已达到当前可用标准。进入 M17 前，仍有两个基础问题需要先收口：

1. 当前显示模式存在不一致，深色模式下顶部栏、控件、字体、表格、卡片、详情栏仍有不少硬编码浅色样式。
2. 本地素材页已经能展示剪切产出，但还没有按项目组织，剪辑师在当前项目内混剪时需要更直接地看到当前项目素材。

M16.4 的目标是在进入 M17 效率优化前，统一 UI 主题系统和本地素材页的信息架构。

## 产品目标

1. 深色模式成为剪辑端默认显示模式，符合长时间看视频、看文案和处理任务的工作场景。
2. 显示模式只保留三个用户选项：`深色`、`浅色`、`系统`。
3. 删除旧的 `默认` 和 `舒适` 概念，避免维护三套视觉逻辑。
4. 左下角用户名区域增加快速主题切换：`Allen  深色 | 浅色 | 系统`。
5. 设置页保留显示模式配置，作为完整配置入口。
6. 本地素材页默认进入“当前项目素材”。
7. 本地素材页提供 `当前项目` / `全部素材` 切换。
8. 全部素材视图按项目分组，项目内按剪切顺序编号排序。
9. 视频封面、播放器、素材画面不应用滤镜或色彩修饰，避免影响剪辑判断。

## 非目标

M16.4 不做完整日志系统、不做新的用户数据面板、不做素材批量删除、不做素材跨项目移动、不做任务优先级、不做 M17 的剪辑效率功能。M16.4 只处理主题系统、本地素材组织和必要的视觉一致性修复。

## 主题策略

### 用户可选项

显示模式保留三个选项：

- `深色`：强制使用深色主题。
- `浅色`：强制使用浅色主题。
- `系统`：跟随系统外观，使用 `prefers-color-scheme` 在深色和浅色之间切换。

### 默认值

产品首次默认显示模式为 `深色`。

历史值迁移规则：

- `dark` -> `dark`
- `light` -> `light`
- `comfort` -> `light`
- `default` -> `dark`
- 空值或无法识别的值 -> `dark`

### 视觉 token

系统只维护两套真实视觉 token：

- 深色 token：主视觉，最高完成度。
- 浅色 token：备用工作台主题，中性专业浅色，不沿用暖色舒适版。

`系统` 不拥有第三套颜色，只根据系统外观选择深色或浅色 token。

### 深色主题原则

深色主题不是浅色主题反转，而是剪辑工作台主视觉。需要完整覆盖：

- App 背景
- 顶部栏
- 侧边栏
- 搜索框
- 按钮
- Tabs / segmented control
- 表格
- 素材卡片
- 任务列表
- 右侧详情栏
- 设置表单
- 弹窗
- 状态标签

深色主题要保持高信息密度和低视觉噪音。背景不使用大面积装饰、渐变光斑或氛围图。主操作色保持克制，状态色需要在暗色背景下重新定义，不能直接套用浅色状态色。

### 浅色主题原则

浅色主题是备用专业工作台，不是暖色舒适模式。它应该保持：

- 中性浅灰画布
- 白色或近白 surface
- 清晰边界
- 蓝色主操作
- 克制状态色
- 与深色主题一致的布局、密度和组件形态

## 左下角主题切换

左下角用户名控件保持轻量，不再弹出用户面板。

结构建议：

```text
Allen
深色 | 浅色 | 系统
```

如果侧边栏宽度不足，可以压缩为同一行：

```text
Allen    深色 | 浅色 | 系统
```

交互规则：

1. 当前选中项使用 filled 或高亮底色。
2. 未选中项使用 text/ghost 状态。
3. 点击任一项立即切换主题并持久化。
4. 控件不得影响本机引擎状态区的可读性。
5. 控件不打开抽屉、不弹出用户数据面板。

设置页里的显示模式仍使用完整文案，用于清晰配置。

## 本地素材页信息架构

### 默认视图

进入本地素材页时，默认显示 `当前项目素材`。

当前项目素材定义为：

- 属于当前选中项目的剪切产出。
- 项目内按剪切顺序编号升序排列。
- 保留横版 / 竖版筛选。
- 保留搜索入口，搜索只在当前项目素材内过滤。

如果当前没有项目，或当前项目没有本地素材，页面显示空状态，并提供切换到 `全部素材` 的入口。

### 全部素材视图

`全部素材` 展示本机所有可复剪素材，按项目分组：

1. 当前项目优先。
2. 其他项目按最近更新时间排序。
3. 未归属项目的素材放入 `未归属素材` 分组。

每个项目组内按剪切顺序编号升序排列。搜索时仍保留分组，但只显示命中素材；素材卡片和详情栏必须显示所属项目。

### 页面切换

页面顶部使用 segmented control：

```text
当前项目 | 全部素材
```

这个切换是本地素材页的核心视图切换，不应藏在筛选菜单里。

### 素材详情

右侧详情栏继续使用 `素材详情`，顶部播放器可播放当前选中的本地素材。点击左侧任一素材卡片，右侧详情同步切换。

详情至少展示：

- 素材名
- 所属项目
- 来源素材
- 时间段
- 时长
- 分辨率
- 横版 / 竖版
- 视频路径或打开文件夹动作
- 完整文案

## UI 规范收口范围

M16.4 需要把主题 token 应用到下列页面：

1. 项目首页
2. 素材定位
3. 剪切任务
4. 本地素材
5. 公共素材库
6. 设置

重点清理硬编码颜色：

- `#fff`
- `#111`
- 浅色背景类 `#f7f8fa`、`#eef6ff`、`#fff5f5`
- 固定 `rgba(255, 255, 255, ...)`
- 固定浅色边框
- 固定浅色状态标签

允许保留素材保护相关颜色：

- 视频播放器黑底
- 封面图本身
- 搜索命中高亮色，但需要为深色主题定义可读版本

## 数据与状态影响

主题状态由现有 appearance 状态承接，但枚举应收敛为：

```ts
type AppearanceMode = "dark" | "light" | "system";
```

本地素材页需要能从素材数据中获得项目归属。优先使用现有剪切任务、manifest 或本地素材元数据中的项目字段；如果缺失，则归入 `未归属素材`，不要丢弃素材。

项目切换后，本地素材页的 `当前项目` 视图应自动跟随当前项目变化。

## 错误处理

1. 主题值异常：回退到 `dark`。
2. 系统外观监听失败：`system` 回退到 `dark`。
3. 当前项目无素材：显示空状态，不报错。
4. 素材缺失项目字段：归入 `未归属素材`。
5. 视频文件不可访问：卡片仍展示元数据，详情栏提示文件不可访问。

## 测试策略

1. appearance 状态测试：
   - 默认值为 `dark`。
   - 旧值 `default` 迁移到 `dark`。
   - 旧值 `comfort` 迁移到 `light`。
   - `system` 能根据系统外观解析到深色或浅色。

2. 页面渲染测试：
   - 设置页只展示 `深色`、`浅色`、`系统`。
   - 左下角用户名区域展示 `Allen  深色 | 浅色 | 系统`。
   - 用户名区域点击主题不会打开用户面板。

3. 本地素材页测试：
   - 默认进入当前项目素材。
   - 可切换到全部素材。
   - 全部素材按项目分组。
   - 当前项目内按剪切顺序排序。
   - 未归属素材进入 `未归属素材` 分组。

4. 视觉回归：
   - 深色模式检查设置页、素材定位、本地素材、剪切任务、公共素材库、项目首页。
   - 浅色模式检查同样页面。
   - 视频封面和播放器不被主题滤镜影响。

## 验收标准

1. 新用户默认进入深色模式。
2. 设置页显示模式只有 `深色`、`浅色`、`系统`。
3. 左下角用户名区域可以直接切换三种主题选择。
4. 旧的 `默认` 和 `舒适` 不再出现在 UI 中。
5. 深色模式下顶部栏、控件、文字、卡片、表格、详情栏无明显浅色残留。
6. 本地素材页默认显示当前项目素材。
7. 本地素材页可以切换到全部素材，并按项目分组。
8. 当前项目素材按剪切顺序编号排列。
9. 所有测试、类型检查和 `build:cutter-web` 通过。
