# M12.2 预处理流水线可观测性设计

日期：2026-05-03
范围：管理端预处理页、仪表盘智能建议、Admin API 预处理队列快照

## 1. 背景

M12.1 已经把管理端主路径收敛为“智能扫描 + 启动预处理流水线 + 自动增量发布”。但真实处理几十 GB 乃至 35T 素材时，管理员仍需要知道系统正在做什么、多久能完成、哪里有风险，以及单个失败是否会拖住全局。

M12.2 的目标是把预处理页从“状态列表”升级为“生产监控台”。

## 2. 用户目标

管理员启动预处理流水线后，应能在一个页面回答这些问题：

- 当前是否在运行。
- 当前正在处理哪个视频。
- 当前视频处于哪个阶段。
- 已经用了多久，预计还剩多久。
- 队列有多少、排到第几、预计什么时候开始处理。
- 今天完成多少、失败多少、整体完成比例如何。
- CPU、内存、磁盘、网络是否会影响继续处理。
- 失败视频为什么失败，以及是否可以重试。

## 3. 产品范围

### 3.1 本阶段做

- 后端预处理任务快照增加中文可观测字段：状态文案、阶段文案、队列位置、预计开始、预计完成、预计剩余。
- 后端根据 `claimed_at`、`completed_at`、`indexed_at`、平均处理耗时计算真实 elapsed 和 estimate。
- 预处理页增加“当前处理视频”区域。
- 预处理页增加“流水线总览”区域，展示整体进度、今日完成、预计完成、自动发布状态。
- 队列列表不展示内部英文 stage、日志路径或无意义的 `0%`。
- 运行负荷异常时，智能扫描和预处理页给出中文建议，例如降低并发或暂停流水线。
- 旧的 `#/index-publish` 继续兼容到预处理页。

### 3.2 本阶段不做

- 后台定时自动扫描。
- 分布式多机预处理。
- 单视频暂停、跳过、优先级调整。
- 复杂图表或历史趋势数据库。
- 剪辑端 UI 重构。

## 4. 数据模型

`AdminPreprocessJob` 扩展以下字段：

- `status_label`：中文状态，例如`正在处理`、`等待处理`、`已完成`、`失败可重试`。
- `stage_label`：中文阶段，例如`提取音频`、`语音识别`、`生成关键帧`、`自动发布索引`。
- `queue_position`：排队位置。非队列任务为 `0`。
- `estimated_start_at`：预计开始时间。无估算时为空字符串。
- `estimated_done_at`：预计完成时间。无估算时为空字符串。
- `estimated_remaining_ms`：预计剩余耗时。

`AdminPreprocessJobsResponse` 扩展 `observability`：

- `running_job_id`
- `running_source_video_id`
- `pipeline_progress_percent`
- `estimated_all_done_at`
- `estimated_queue_duration_ms`
- `throughput_label`
- `load_advice`

这些字段只服务管理端可观测，不改变协议层 manifest 的可见性规则。

## 5. 页面设计

预处理页主区顺序：

1. 生产状态：一句话说明当前系统状态。
2. 流水线总览：整体进度、预计完成、今日完成、失败数量、自动发布状态。
3. 当前处理视频：当前视频、阶段、进度、已用时、预计剩余、预计完成。
4. 素材来源和索引状态。
5. 正在处理、队列中、最近完成、失败可重试。

右侧控制区：

- 队列参数。
- 阶段说明。
- 预处理服务。
- 运行负荷。
- 负荷建议。
- 启动/暂停流水线。

## 6. 估算规则

- 平均处理耗时优先使用仪表盘 production 的 `average_video_process_ms`。
- 如果没有平均值，使用当前视频时长或显示`等待更多样本`。
- 队列第 N 个视频的预计开始时间 = 当前时间 + 前面等待数量 * 平均耗时 / 并发任务数。
- 队列预计完成时间 = 当前时间 + 队列剩余数量 * 平均耗时 / 并发任务数。
- 当前处理任务的剩余时间 = 平均耗时 - 已用时，最小为 0。

## 7. 风险提示

运行负荷建议使用中文摘要：

- CPU 或内存 blocked：建议暂停流水线或降低并发。
- 磁盘 blocked：建议停止预处理并清理空间。
- 网络 blocked：建议暂停并检查网络或语音识别服务。
- attention：建议继续观察，必要时降低并发。
- healthy：显示`运行负荷正常，可以继续处理`。

## 8. 验收标准

- 预处理页能看到`流水线总览`、`当前处理视频`、`预计剩余`、`预计完成`、`负荷建议`。
- 队列中视频显示排队位置和预计开始，不显示内部 stage、日志路径或无意义 `0%`。
- 正在处理视频的 elapsed、estimated remaining、progress 由后端快照计算。
- 智能扫描在运行负荷 blocked 时优先提示处理系统负荷风险。
- 用户可见文案全部中文。
- `npm run typecheck` 与 `npm test` 通过。
