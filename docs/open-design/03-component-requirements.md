# MixLab Open Design 组件需求

本文档用于 Open Design 生成 Web App UI 原型时定义组件范围。组件说明面向设计，不是 React 实现规范。

## 全局布局组件

### 组件用途

承载管理端和剪辑端的应用框架，包括侧边栏、主内容区、右侧 Inspector、局部刷新状态和响应式布局。

### 出现场景

- 管理端全部页面。
- 剪辑端首页、素材搜索、公共素材库、原视频详情、剪切任务、本地素材、设置。

### 包含内容

- App Shell。
- Sidebar。
- Main Content。
- Optional Inspector / Detail Panel。
- Page Header。
- Status / Notice 区。

### 交互状态

- 初始加载。
- 局部刷新。
- 页面错误。
- 当前路由高亮。
- Inspector 展开/收起：建议补充。

### 视觉层级

- 一级：导航和页面标题。
- 二级：主任务区域。
- 三级：辅助状态、详情、诊断。

### 设计注意事项

- 不要做传统后台顶部大导航 + 左侧二级菜单堆叠。
- 工具型应用应稳定、紧凑、可扫描。
- 右侧 Inspector 不应挤压主工作流到不可用。

## Sidebar

### 组件用途

承载端内主导航和运行摘要。

### 出现场景

- 管理端主导航。
- 剪辑端主导航。

### 包含内容

- 导航项图标。
- 导航项名称。
- 当前路由高亮。
- 剪辑端 footer：公共素材数量、本地素材数量、剪切任务状态、本机服务状态、当前用户。

### 交互状态

- 默认。
- hover。
- active。
- disabled：需要确认是否有权限隐藏场景。
- collapsed：建议补充。

### 视觉层级

- 当前页面高亮明显但克制。
- Footer 状态低于导航，不抢主视觉。

### 设计注意事项

- 图标要清晰，不要使用抽象字符作为最终设计；建议使用专业线性图标。
- 中文标签宽度要稳定。

## Header

### 组件用途

展示当前页面标题、说明、关键操作或上下文。

### 出现场景

- 管理端 `AdminPageHeader`。
- 剪辑端各页面 `cutter-page-header`。
- 素材搜索命令区。

### 包含内容

- Eyebrow。
- H1 页面标题。
- 页面说明。
- 主操作按钮或搜索框。
- 状态条：可选。

### 交互状态

- 默认。
- loading：标题可先显示，状态条 skeleton。
- error：可展示全局错误入口。

### 视觉层级

- 标题是页面 anchor。
- 操作只保留核心动作。

### 设计注意事项

- 不要把多个危险动作放入 Header。
- Header 高度要稳定，不因数据刷新跳动。

## Navigation

### 组件用途

定义端内页面跳转和上下文跳转。

### 出现场景

- 主 Sidebar。
- 项目切换器。
- 面包屑或返回入口：建议补充。
- 表格行详情跳转。

### 包含内容

- 当前 route。
- 可跳转 route。
- 兼容路由不直接展示。

### 交互状态

- active。
- hover。
- focus。
- disabled。

### 视觉层级

- 主导航一级。
- 详情/返回二级。

### 设计注意事项

- 管理端不建议把 Doctor 当主导航高频入口，除非产品确认。
- 原视频详情和剪辑端原视频详情都应从上下文进入。

## Card

### 组件用途

展示单个指标、状态、素材、项目、任务摘要或设置检查。

### 出现场景

- 仪表盘 KPI。
- 下一步建议。
- 核心链路健康。
- 项目卡片。
- 预处理流水线阶段。
- 本机剪切流水线。
- Gallery item。

### 包含内容

- 标题。
- 主数值或主状态。
- 说明文本。
- 可选图像/封面。
- 可选状态标签。
- 可选操作。

### 交互状态

- default。
- hover。
- selected。
- processing。
- warning。
- failed。
- disabled。

### 视觉层级

- 单卡片内部只允许一个主数值或主动作。
- 多卡片区域保持一致高度。

### 设计注意事项

- 卡片圆角不超过 8px。
- 不要卡片套卡片。
- 不要大面积渐变卡。

## Table

### 组件用途

展示高密度结构化数据：原视频、任务、索引版本、用户、检查项。

### 出现场景

- 原视频管理。
- 预处理任务队列。
- 索引版本。
- 剪辑师用户。
- 剪切任务。
- Doctor 检查项。

### 包含内容

- 表头。
- 行。
- 状态标签。
- 行主标题。
- 次要元数据。
- 行操作。
- 分页或加载更多。

### 交互状态

- default。
- hover。
- selected。
- loading row。
- empty。
- error。
- disabled action。

### 视觉层级

- 第一列或标题列为扫描锚点。
- 状态列用短标签。
- 操作列低调，避免每行按钮太抢。

### 设计注意事项

- 长路径、长文案必须截断并可展开/复制。
- 表格行高度稳定，局部刷新不跳动。
- 首屏数据不能一次性加载过多。

## Form

### 组件用途

承载设置、公开元数据、项目命名、登录申请、筛选和偏好配置。

### 出现场景

- 管理端设置。
- 原视频 Inspector 元数据编辑。
- 剪辑端设置。
- 登录申请。
- 项目新建/重命名。

### 包含内容

- Label。
- Input。
- Select。
- Textarea。
- Checkbox / Toggle。
- Radio。
- 保存 / 取消。
- 验证消息。

### 交互状态

- default。
- focus。
- dirty。
- saving。
- success。
- error。
- disabled。

### 视觉层级

- 表单分组清楚。
- 保存按钮只在必要区域出现。

### 设计注意事项

- 密钥输入只显示配置状态，不能明文展示。
- 路径字段要适合长文本。
- 危险表单项必须有确认。

## Modal

### 组件用途

处理明确的短流程和高风险确认。

### 出现场景

- 新建项目。
- 重命名项目。
- 删除项目。
- 停用剪辑师用户。
- 建议补充：发布索引、恢复卡住任务、批量重试失败任务确认。

### 包含内容

- 标题。
- 说明。
- 输入或选项。
- 取消按钮。
- 主按钮或危险按钮。

### 交互状态

- open。
- submitting。
- validation error。
- disabled。
- destructive。

### 视觉层级

- 模态中的主动作明确。
- 危险动作使用 danger 色，不使用普通主色。

### 设计注意事项

- 删除项目必须明确“公共素材库源视频不会被删除”。
- 停用用户必须说明登录凭证失效。
- 不要把长表单放入 Modal。

## Drawer

### 组件用途

承载可临时查看但不打断主工作流的详情或日志。

### 出现场景

- 建议补充：预处理任务日志。
- 建议补充：原视频产物详情。
- 建议补充：用户设备详情。
- 建议补充：Doctor 技术详情。

### 包含内容

- 标题。
- 状态。
- 详情字段。
- 操作区。

### 交互状态

- open / close。
- loading。
- error。
- content loaded。

### 视觉层级

- Drawer 高于 Inspector，但低于危险确认 Modal。

### 设计注意事项

- 日志内容使用等宽字体，但不应污染普通页面。

## Button

### 组件用途

触发页面、表单、任务和导航动作。

### 出现场景

全站。

### 包含内容

- Label。
- Optional icon。
- Loading spinner：建议补充。
- Tooltip / reason：当前代码用 title 表达原因。

### 交互状态

- primary。
- secondary。
- tertiary / inline。
- danger。
- disabled。
- loading。
- success feedback。

### 视觉层级

- 每个区域最多一个 primary。
- 危险按钮单独样式。

### 设计注意事项

- 不要把所有动作都做成同等按钮。
- 对于工具按钮，优先用图标 + tooltip。
- 文案短、明确、动词开头。

## Badge

### 组件用途

展示数量、类型、属性和轻量标记。

### 出现场景

- 项目卡片：已剪、搜索。
- 素材卡片：横版/竖版、标签。
- 任务筛选 tab 数量。
- 用户状态统计。

### 包含内容

- 短文本。
- 可选数字。
- 可选色彩语义。

### 交互状态

- default。
- active。
- warning。
- failed。

### 视觉层级

- Badge 不应比主标题更抢眼。

### 设计注意事项

- 标签过多时折叠或限制数量。

## Status Tag

### 组件用途

表达业务状态。

### 出现场景

- 预处理状态。
- 任务状态。
- 用户状态。
- Doctor 状态。
- 运行负荷。
- 搜索连接状态。

### 包含内容

- 状态文本。
- 色彩。
- 可选图标/点。
- 可选说明 tooltip。

### 交互状态

- ready / healthy。
- processing。
- queued。
- attention / warning。
- blocked / failed。
- disabled。

### 视觉层级

- 状态标签要短，详情放在旁边或 tooltip。

### 设计注意事项

- 不能只靠颜色表达状态，必须有文字。
- “处理中但服务未运行”应显示为“待恢复”，不能误导为运行中。

## Empty State

### 组件用途

在没有数据或用户还未开始操作时引导下一步。

### 出现场景

- 无项目。
- 未搜索。
- 无候选素材。
- 无本地素材。
- 无预处理任务。
- 无剪辑师申请。
- 无匹配原视频。

### 包含内容

- 标题。
- 简短说明。
- 可选主动作。
- 可选链接。

### 交互状态

- neutral。
- search-empty。
- no-permission。
- not-initialized。

### 视觉层级

- 空状态比普通说明更显眼，但不能像营销卡。

### 设计注意事项

- 必须告诉用户下一步，不要只说“暂无数据”。

## Loading State

### 组件用途

表达首屏、局部面板、分页、搜索和任务刷新。

### 出现场景

- 管理端仪表盘局部卡片。
- 原视频首批加载和继续加载。
- 预处理任务明细。
- 原视频详情。
- 剪辑端搜索。
- 公共素材库继续加载。
- 任务刷新。

### 包含内容

- Skeleton。
- Inline loading label。
- Progress / meter。
- aria-live 文案。

### 交互状态

- first load。
- partial refresh。
- load more。
- background loading。

### 视觉层级

- 页面 shell 先显示，局部内容加载。

### 设计注意事项

- 避免整页闪烁。
- 加载占位尺寸要稳定。
- 搜索可先显示首批结果，再显示“继续加载中”。

## Error State

### 组件用途

清楚表达服务不可用、数据加载失败、操作失败、任务失败和环境检查失败。

### 出现场景

- 管理端服务连接失败。
- 原视频详情失败。
- 搜索失败。
- 剪切任务失败。
- Doctor fail。
- 设置保存失败。

### 包含内容

- 用户可读标题。
- 原因说明。
- 重试/返回/检查入口。
- 详细技术信息折叠：诊断页可展开。

### 交互状态

- recoverable。
- blocked。
- permission denied。
- network/server failure。

### 视觉层级

- 错误要足够明显，但不应让整个应用失控。

### 设计注意事项

- 普通页面不要显示 raw stack trace。
- 技术术语只在诊断详情中出现。

## Toast

### 组件用途

反馈短操作结果。

### 出现场景

- 保存设置完成。
- 元数据保存完成。
- 封面保存完成。
- 扫描完成。
- 剪切提交完成。
- 复制文案完成。
- 操作失败。

### 包含内容

- 成功/失败/提示图标。
- 短消息。
- 可选动作：查看任务、撤销需要确认。

### 交互状态

- info。
- success。
- warning。
- error。

### 视觉层级

- 不遮挡核心操作区。

### 设计注意事项

- 当前代码有 actionNotice / actionError，设计可统一为 Toast + 页面内局部 notice。

## Detail Panel

### 组件用途

在不离开当前页面时展示选中对象详情和次要操作。

### 出现场景

- 原视频管理右侧素材详情。
- 公共素材库资源信息。
- 本地素材详情。
- 剪切任务详情。
- 管理端仪表盘健康摘要。
- 管理端设置系统状态。

### 包含内容

- 标题。
- 选中对象封面/预览。
- 关键字段。
- 状态。
- 次要操作。

### 交互状态

- no selection。
- selected。
- loading。
- error。
- editable。
- read-only。

### 视觉层级

- 主内容优先，Detail Panel 辅助。

### 设计注意事项

- 不要把所有操作塞入详情面板；只放与当前对象相关的动作。

## Search Box

### 组件用途

支持原视频搜索、剪辑端文案搜索和项目入口搜索。

### 出现场景

- 管理端原视频管理。
- 剪辑端首页。
- 剪辑端素材搜索。

### 包含内容

- Search icon。
- Input。
- Placeholder。
- Submit button。
- Clear button。
- Optional recent searches：建议补充。

### 交互状态

- idle。
- typing。
- submitting。
- searching。
- empty result。
- error。

### 视觉层级

- 剪辑端首页和素材搜索的搜索框是主入口。
- 管理端原视频搜索是工具栏级别。

### 设计注意事项

- 文案搜索支持关键词或粘贴长文案，输入框应支持较长文本。
- 搜索中不应阻塞已有结果。

## Filter Bar

### 组件用途

帮助用户快速切换数据范围。

### 出现场景

- 原视频状态筛选。
- 公共素材横竖版筛选。
- 本地素材当前项目/全部素材。
- 本地素材横竖版筛选。
- 剪切任务状态筛选。
- 建议补充：素材搜索公共/本地来源筛选。

### 包含内容

- Chip / segmented buttons。
- 数量。
- active 状态。
- select fallback：窄屏。

### 交互状态

- default。
- active。
- disabled。
- loading count。

### 视觉层级

- 筛选是二级操作，不能比页面主动作更重。

### 设计注意事项

- 计数刷新不要导致按钮宽度跳动。
- `aria-pressed` 状态要清楚。

## Timeline / Progress 组件

### 组件用途

表达预处理、剪切、索引发布等多阶段流程。

### 出现场景

- 管理端预处理流水线。
- 管理端当前任务阶段进度。
- 剪辑端本机剪切流水线。
- 剪切任务 phase timings。
- 桌面首启步骤：需要确认是否纳入本次。

### 包含内容

- 阶段名称。
- 阶段序号。
- 当前状态。
- 进度百分比或 meter。
- 时间：开始、耗时、预计完成。

### 交互状态

- pending。
- running。
- done。
- failed。
- paused/stale。

### 视觉层级

- 当前阶段最显眼，历史和后续阶段次级。

### 设计注意事项

- 当服务未运行但任务仍标记 processing 时，用“待恢复”表达。
- Progress 不应伪造精度；无数据时显示等待估算。

## 项目中特有的业务组件

## Admin Smart Scan / 下一步建议

### 组件用途

聚合系统状态，给管理员一个最重要的下一步动作。

### 出现场景

- 管理端仪表盘。

### 包含内容

- severity。
- title。
- detail。
- primary action。
- suggestions。

### 交互状态

- healthy。
- attention。
- blocked。
- action running。

### 视觉层级

- 仪表盘主行动卡。

### 设计注意事项

- 一次只突出一个主按钮。
- 建议文本必须用户化。

## Admin Core Path Health / 核心链路健康

### 组件用途

展示从搜索到文案再到剪切的核心路径是否可用。

### 出现场景

- 管理端仪表盘。

### 包含内容

- 总状态。
- 关键词定位。
- 完整文案。
- 选段剪切。
- 50 人容量。

### 交互状态

- healthy。
- attention。
- blocked。

### 视觉层级

- 重要但不应比下一步建议更抢。

### 设计注意事项

- 如果显示 blocked，必须直接解释是哪一行造成，不要只给红色状态。
