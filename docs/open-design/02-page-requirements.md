# MixLab Open Design 页面需求

本文档用于 Open Design 生成完整 Web App UI 原型。请优先设计桌面端。所有页面都应体现真实业务状态、真实数据密度和局部加载，不要做营销落地页。

## 信息架构

MixLab 是双端产品：

- 管理端：公共素材库生产与系统管理。
- 剪辑端：项目化素材搜索、选段、剪切与本地素材复用。

建议 Open Design 输出同一套设计系统下的两个 Web App Shell：

- Admin Shell：浅色、高密度、管理视角，重状态与生产链路。
- Cutter Shell：可浅色或深色，偏工作台，重搜索、文案、视频和剪切效率。

需要确认：Open Design 是否应该把管理端和剪辑端画成两个独立入口，还是同一个原型内用模式切换。

## 管理端仪表盘

### 页面目标

帮助管理员快速判断公共素材库是否能支撑剪辑师工作，并明确下一步应该扫描、预处理、重试、恢复、发布索引还是检查系统。

### 主要用户问题

- 当前素材库有多少可搜索素材？
- 预处理是否在正常生产？
- 剪辑端搜索到剪切的主链路是否健康？
- 是否有失败、待恢复、待发布或系统环境问题？
- 我下一步要点什么？

### 页面内容模块

- 页面标题与公共库状态条。
- KPI 网格：可搜索总时长、可用视频、句子片段、当前索引、失败任务、活跃剪辑师。
- 下一步建议卡片。
- 核心链路健康卡片。
- 生产吞吐可视化。
- 生产健康列表。
- 详细指标分组：素材规模、文案与索引、预处理产能、剪辑端使用、转化漏斗、风险摘要、系统负荷。
- 磁盘空间。
- 最近任务。
- 最近预警。
- 右侧 Inspector：公共素材库健康摘要、索引状态。

### 需要展示的数据

- 公共库根目录：路径，建议以可折叠或中间省略显示。
- 当前索引：版本号；普通视图可显示为“已发布版本”，详情中显示原始版本。
- 发布状态：ready / building / needs-publish / error。
- 最近更新：时间。
- 可搜索总时长、总时长、可用视频比例。
- 文案片段数、文案视频数、当前索引。
- 失败任务数、活跃剪辑师数、搜索请求、搜索 p95、搜索覆盖。
- 预处理 supervisor 状态、 active / queued / failed / completed。
- Doctor pass / warn / fail。
- 磁盘 total / available。

### 主要操作

- 扫描新增素材。
- 执行下一步建议，例如启动预处理、重试失败、恢复卡住任务、运行系统检查。
- 建议补充：从各风险行直接跳转到对应页面和筛选状态。

### 页面状态

- empty：首次未初始化或暂无素材，展示初始化/设置入口。
- loading：首屏 shell + skeleton，KPI 区块局部加载。
- error：管理端服务不可连接或仪表盘数据加载失败，提供重试和检查服务提示。
- success：素材库健康，无待办，显示“暂无待办”。
- processing：预处理运行中，只刷新相关卡片和任务摘要。
- disabled：高风险动作在权限不足或服务不可用时 disabled，并给出原因。

### 交互说明

- KPI 和健康行可点击进入详情页或带筛选的列表页。
- 下一步建议只保留一个主操作；其他建议以轻量列表呈现。
- 仪表盘不做整页定时刷新；每个卡片局部刷新，并显示更新时间或刷新中状态。
- 最近任务点击进入预处理页面对应任务。

### 设计注意事项

- 避免“按钮墙”。仪表盘最多一个强主按钮。
- “核心链路健康”要用用户能理解的语言解释原因，不只显示“链路阻塞”。
- 技术术语如 searchd、SQLite、fallback、index_version 不应在普通仪表盘主视觉出现。
- 高密度但要有扫描路径：先状态，再风险，再详情。

## 原视频管理

### 页面目标

帮助管理员浏览、筛选、检查和维护公共素材库中的原视频，并对单条视频执行必要的预处理或发布操作。

### 主要用户问题

- 原视频是否已经被系统发现？
- 哪些视频可搜索、排队中、失败、未处理、待发布？
- 某个视频的公开信息是否正确？
- 某个失败或未处理视频应该如何处理？

### 页面内容模块

- 页面标题和搜索框。
- 公共素材来源状态条：素材来源、全部原视频、可搜索、预处理状态、搜索可见、总时长、已载入。
- 状态筛选条：全部、可用、处理中/待恢复、队列中、未处理、失败、待发布。
- 原视频表格。
- 分页/继续加载。
- 右侧素材详情 Inspector：封面、状态、公开元数据、封面上传、单条处理动作。

### 需要展示的数据

- 封面：`cover_url`。
- 标题：`title`，为空时显示 `file_name`。
- 编号：`source_video_id`。
- 文件名：`file_name`。
- 时长：`duration_ms`。
- 相对路径：`relative_path`。
- 字幕状态：已生成 / 生成中 / 等待 / 失败 / 待恢复。
- 预处理状态：`preprocess_status`。
- 搜索可见：`visible_to_cutters`。
- 发布版本：当前索引 / 待发布 / 空。
- 公开元数据：标题、标签、说明、讲师、课程、分类。
- 错误信息：`error_stage`、`error_message`。

### 主要操作

- 搜索文件名 / 标签 / 相对路径。
- 按预处理状态筛选。
- 选择列表行。
- 查看详情。
- 上一页 / 下一页。
- 继续加载更多。
- 保存素材信息。
- 保存封面。
- 加入预处理 / 重新处理 / 恢复到队列 / 发布到剪辑端。

### 页面状态

- empty：没有匹配原视频。
- loading：首批 20 条加载中；更多分页加载中。
- error：列表加载失败、保存失败、封面格式不支持。
- success：保存完成、队列操作完成。
- processing：单条视频处理中；状态显示生成中或待恢复。
- disabled：无权限或接口不可用时，写入动作 disabled。

### 交互说明

- 列表初始只加载小批量，代码当前首批为 20 条。
- 搜索和筛选应支持服务端结果；若只筛已载入数据，需要清楚说明范围。
- 行点击选中并刷新 Inspector，查看详情进入原视频详情页。
- 封面上传只接受 JPG / PNG / WebP。
- 写入动作必须二次确认或明确状态，尤其是发布、恢复、重新处理。

### 设计注意事项

- 主列表要稳定，不因 Inspector 编辑导致表格抖动。
- 路径很长，必须中间省略且可复制。
- 状态筛选不要占过多高度；大屏可 chip，小屏可 select。
- 不要把“安全开关/解锁”作为页面主要概念；写入风险通过确认弹窗和权限状态表达。

## 原视频详情

### 页面目标

让管理员查看单条原视频的完整预处理数据、产物完整性、技术信息和剪辑师可见原因。

### 主要用户问题

- 这条视频为什么可见或不可见？
- 文案、字幕、封面、关键帧、索引是否完整？
- 失败发生在哪个阶段？
- 原视频的技术信息和公开元数据是什么？

### 页面内容模块

- 页面标题。
- 封面预览。
- 基本信息。
- 技术信息。
- 预处理状态。
- 产物完整性。
- 文案数据。
- 视觉数据。
- 公开元数据。
- 右侧剪辑师可见状态。

### 需要展示的数据

- 原视频编号、标题、文件名、相对路径、更新时间。
- 时长、分辨率、帧率、编码、文件大小、内容哈希。
- 状态、当前阶段、任务编号、尝试次数、开始/完成/失败时间、失败阶段、失败原因。
- 文案、字幕、封面、关键帧的状态和路径。
- 索引版本。
- 完整文案、分段数量、字符数量。
- 公开标题、标签、说明、讲师、课程、分类。
- 对剪辑师可见、状态说明、原因。

### 主要操作

- 当前代码主要为只读查看。
- 建议补充：返回原视频管理、复制路径、打开对应预处理任务日志、从详情进入重试/恢复/发布确认。

### 页面状态

- loading：正在读取原视频详情。
- empty：没有可查看的原视频。
- error：原视频不存在、接口不可用、加载失败。
- success：详情完整显示。
- disabled：只读详情不显示危险操作，或把操作放入二级区域。

### 交互说明

- 从原视频管理点击“查看详情”进入。
- 如果详情失败，应提供返回原视频管理。
- 文案较长时应折叠或使用可滚动区域，不要撑爆页面。

### 设计注意事项

- 这是排障页，允许比普通页多一些技术细节。
- 仍要把错误信息转换成用户能理解的文本。
- 产物完整性适合做 checklist 或 status table。

## 预处理

### 页面目标

帮助管理员理解并控制公共素材从原视频到可搜索索引的生产流水线。

### 主要用户问题

- 预处理服务是否运行？
- 当前在处理哪条视频、哪个阶段？
- 队列还有多少，预计什么时候完成？
- 哪些失败或卡住，需要我处理？
- 哪些视频已完成但还没有进入剪辑端搜索？

### 页面内容模块

- 页面标题与预处理状态条。
- 生产状态卡片。
- 指标带：未处理、将加入、预计总时长、正在处理/待恢复、队列中、最近完成、失败可重试。
- 流水线阶段：扫描素材、提取音频、语音识别、生成文案、封面关键帧、发布索引。
- 流水线总览：整体进度、预计完成、今日完成、负荷建议。
- 当前处理视频卡片。
- 任务队列表格。
- 索引发布区域。
- 素材来源与索引状态摘要。
- 右侧处理控制和任务日志。

### 需要展示的数据

- Supervisor：服务状态、当前任务、当前索引、预计完成。
- Observability：整体进度、预计全部完成、吞吐说明、负荷建议。
- 当前任务：source id、标题、状态、阶段、进度、已用时、预计剩余、预计完成。
- 任务队列：任务 id、原视频、阶段、排队位置、进度、耗时、预计开始、预计完成、操作。
- 索引：当前索引、可搜索视频数、校验状态、待发布视频数、自动增量发布状态、系统检查摘要、历史版本。
- 上次处理结果：领取、成功、失败。

### 主要操作

- 启动预处理。
- 暂停预处理。
- 重试失败视频。
- 恢复卡住任务。
- 查看任务处理详情/日志。
- 发布到剪辑端。
- 建议补充：按状态筛选任务、只看失败、只看当前项目不适用于管理端。

### 页面状态

- empty：暂无预处理任务。
- loading：任务明细后台同步中。
- error：任务列表或日志读取失败。
- success：流水线空闲且无待处理，或任务完成。
- processing：服务运行中、任务进行中、索引发布中。
- disabled：不可启动/不可停止时按钮禁用；运行负荷阻塞时启动动作需要提示。

### 交互说明

- 预处理页可以自动刷新，但必须局部刷新任务/进度，不要整页闪。
- 任务行点击或“详情”打开日志面板。
- 当 active_count > 0 但 supervisor 未运行时，应明确显示“待恢复”，不要显示为正常运行。
- 发布索引是影响剪辑端的关键动作，应在索引区域说明结果。

### 设计注意事项

- 把复杂流水线设计成“当前状态 + 阶段时间线 + 任务列表 + 例外处理”。
- “启动预处理”是主路径；重试、恢复、发布应低一层级。
- 不要让日志占据主页面，日志适合 Drawer / Inspector / Modal。

## 系统检查

### 页面目标

帮助管理员或运维检查公共素材库、产物库、索引、工具、ASR 和本地素材等运行条件。

### 主要用户问题

- 系统哪里不满足运行要求？
- 这些检查项影响什么？
- 应该如何修复？
- 是否能导出检查报告？

### 页面内容模块

- 页面标题和重新检查按钮。
- 通过/警告/失败指标。
- 检查项列表。
- 检查结果详情：检查目的、失败影响、处理建议、技术详情。
- 右侧检查报告：生成时间、协议版本、库路径、summary。
- 导出检查报告按钮。

### 需要展示的数据

- Doctor summary：pass / warn / fail。
- 每个 check：check_id、用户化名称、status、message。
- 报告：generated_at、schema_version、library_root。

### 主要操作

- 重新检查。
- 导出检查报告。
- 建议补充：从失败项跳转到设置、预处理或原视频详情。

### 页面状态

- loading：检查中。
- empty：暂无检查结果。
- error：检查失败或报告导出失败。
- success：全部通过。
- processing：正在重新检查。
- disabled：导出不可用或服务不可连接。

### 交互说明

- 默认展示用户化解释；技术详情可折叠。
- 失败项优先排序，其次警告，再通过。

### 设计注意事项

- Doctor 是诊断页，可以显示技术细节，但首层必须是“影响”和“建议”。
- 不建议作为普通用户主导航高频入口，可作为设置/仪表盘进入。

## 剪辑师用户

### 页面目标

让管理员审核剪辑师登录申请、查看设备和使用情况，并停用异常用户。

### 主要用户问题

- 谁在等待审核？
- 哪些用户已通过、已拒绝、已停用？
- 用户最近是否在使用，有没有搜索或剪切失败？
- 如何停用一个用户？

### 页面内容模块

- 页面标题。
- 使用指标：活跃剪辑师、搜索次数、搜索失败、选段次数、剪切成功。
- 审批状态指标：待审核、已通过、已拒绝、已停用。
- 用户表格。
- 右侧用户概览。
- 停用确认弹窗。

### 需要展示的数据

- 用户：display_name、username、status、devices.length、last_used_at。
- 使用指标：search_request_count、search_failure_count、cut_success_count。
- 设备：device_name、status、first_seen_at、last_login_at、ip、user_agent；当前列表只显示设备数，详情建议补充。

### 主要操作

- 通过申请。
- 停用用户。
- 取消/确认停用。
- 建议补充：查看设备详情、按状态筛选用户。

### 页面状态

- empty：暂无剪辑师申请。
- loading：正在读取剪辑师用户。
- error：用户列表或操作失败。
- success：审批成功、停用成功。
- disabled：非 pending 用户不能通过；非 approved 用户无需停用。

### 交互说明

- 停用必须二次确认。
- 审批成功后用户状态即时更新。

### 设计注意事项

- 权限管理应简单，避免复杂后台风格。
- 风险操作放在表格行末和确认弹窗，不要过度突出。

## 管理端设置

### 页面目标

管理公共素材库基础信息、素材来源、预处理参数、ASR 密钥和运行环境。

### 主要用户问题

- 公共素材库名称和路径配置是否正确？
- 素材来源有哪些，是否启用？
- 预处理并发、音频模式和工具是否可用？
- ASR 密钥是否已配置？
- 是否需要初始化素材库？

### 页面内容模块

- 页面标题和新增素材来源按钮。
- 素材库基本信息。
- 素材来源列表。
- 预处理设置。
- 语音识别设置。
- 右侧系统状态/路径检查。
- 初始化素材库提示。
- 保存设置、检查语音识别按钮。

### 需要展示的数据

- library_name。
- source_folders：name、path、enabled、discovered_video_count、new_unprocessed_count、last_scanned_at。
- artifact_library：path、mode、migration_required。
- runtime_policy：audio_mode、concurrent_jobs、auto_scan_enabled、auto_queue_enabled、auto_publish_index_enabled。
