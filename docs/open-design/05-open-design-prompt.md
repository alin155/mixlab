# Open Design 最终 Prompt

复制下面这段给 Open Design：

```text
这是一个 Web App UI 设计任务。请为当前真实项目 MixLab V3 生成完整 Web App UI 原型。

请先阅读并遵守以下文档：

1. docs/open-design/01-product-brief.md
2. docs/open-design/02-page-requirements.md
3. docs/open-design/03-component-requirements.md
4. docs/open-design/DESIGN.md

设计目标：

- 输出完整 Web App UI 原型。
- 优先设计桌面端。
- 覆盖管理端和剪辑端的所有核心页面。
- 管理端包括：仪表盘、原视频管理、原视频详情、预处理、系统检查、剪辑师用户、设置。
- 剪辑端包括：登录/申请、首页/项目入口、素材搜索、公共素材库、原视频详情、剪切任务、本地素材、设置。
- 重点优先设计剪辑端素材搜索工作台、剪辑端首页/项目入口、管理端仪表盘、管理端预处理。
- 所有页面都必须包含真实产品需要的 empty、loading、error、success、processing、disabled 状态。
- 必须遵守 DESIGN.md 的设计系统方向。
- 风格是现代 SaaS Web App，专业、高效、清晰，类似 Linear / Notion / Cursor / Vercel 的工具型产品气质。
- 不要传统后台管理系统风格。
- 不要花哨渐变。
- 不要低级 AI 生成感。
- 不要过度装饰。
- 不要营销落地页。
- 信息层级必须清楚。
- 操作路径必须短。
- 管理端不要做按钮墙；低频和危险动作放入详情、设置、Drawer 或确认弹窗。
- 剪辑端必须围绕项目化搜索、文案选择、视频预览和剪切任务形成高效工作流。
- 大数据页面必须体现分页、加载更多或虚拟滚动，不要一次性渲染大量数据。
- 页面刷新必须是局部刷新，不要设计整页闪烁式刷新。

请不要开发真实业务逻辑，不要写真实前端代码，不要重构项目代码。你只需要输出 UI 原型和设计说明。

请在结果中提供：

1. 完整页面原型。
2. 关键组件设计。
3. 状态设计，包括空、加载、错误、处理中、禁用。
4. 页面之间的跳转关系。
5. 关键交互说明。
6. 可以交给 Codex 后续落地的设计说明，包括布局、组件、状态、响应式和数据密度要求。

如果文档中标注“需要确认”，请在原型或说明中保留为待确认项，不要自行编造。
如果文档中标注“建议补充”，可以作为 UI 原型中的合理增强，但请在说明中标记为建议补充，不要当作当前已实现功能。
```
